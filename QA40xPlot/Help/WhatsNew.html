What's New in QA40xPlot

v 1.4.1    April 12, 2026

Risk Summary

This release mainly works on synchronizing external generators - which now seem to work well. Some slight improvements to the idle read loop. It should be low risk.

Impedance Range Increase

Increase the impedance max to 100,000 ohms on the graph to support larger test impedances.

Brighten Graph Plot Area

On lower brightness monitors the graph plot area was a bit dark so I brightened the default. I strongly recommend you set it the way you like based on monitor brightness and save a default configuration.

File Save and Load with Configuration

The saved file now contains all test configurations as well as all test data. Load now loads all test data and sets the configuration to match the data. When Open-ing a new dialog asks which pieces (data or configuration or both) to load.

Chirp Windowing

FFT WIndowing for chirps was originally a 'why not' but the answer is that windowing chirps changes response curves unless the latency math is perfect. So, a rectangular window is both mathematically better and practically necessary. There's no reason to force a user to pick that so Chirp is now always non-windowed during tests (in the Frequency Response test group and Spectrum). This change substantially increases chirp accuracy at the sweep frequency limits.

v 1.3.1    March 25, 2026

Risk Summary

This release is primarily about improving the robustness of the new idle code and reducing risk.

Improve Chirp Accuracy

The new chirp formatting recaptures blank space leftover from historical reasons. It now uses more of the available time and should be more accurate than previously.

Added History Display

For diagnostic purposes if you turn off the generator in Spectrum or Scope it will let you display any of the last 100 acquisitions in the History list (any non-noise readings). Turn the checkmark on and cycle up or down to watch the display change. Note that the memory storage requirement for this is about 250MB and this will become a setting.

v 1.3.0    March 21, 2026

Risk Summary

This is extremely high risk due to the upgrades required to support continuous generation. Please report any new bugs via github or the forum. Because the changes are down deep in the USB code bugs here can cause issues possibly requiring a power cycle of the QA403 or QA402 tester. These are the last planned high-risk changes for a long while.

Add Continuous Generator Support

There is a new setting Idle Generator which can be on or off. When turned on (the default) any case where the generator repeats (such as averaging or the Spectrum test in Run mode) the generator will run continuously. Prior to this the generator would run as packets, with a short pause between each fft block. Now it just runs until stopped. The waveform will be discontinuous at one sample - junctions between the fft blocks - but the sampled waveform should more or less be identical at each cycle. This may produce more accurate testing results because the device under test has fewer start/stop transients and the tests run subjectively faster because the test cycle doesn't wait for the graph to plot.

Default Color Change

My default color scheme has changed, so I thought it was time to update the default colors to my preferred gray palette. You can always change them in Settings and save that as the new default. The default plot line colors are unchanged, though the dark blue is a bit low-contrast for the scheme.

New Latency Setting

Previously, USB latency (delay until the generator starts sending after we politely ask the USB port) was determined by examining the input data for a signal. This was squirrely when testing noise, which has no data, and with low voltage and low frequency stuff which starts slowly. After some empirical testing with the current code it became clear the device latency is constant and a fixed number of samples... so there is a new advanced setting to set that Latency. To test it, use the Scope test with a sine wave and the sine wave should start near time 0 (within a sample or two).

v 1.2.10    March 7, 2026

Risk Summary

This is extremely high risk due to the upgrades required to support load/save. Please report any new bugs via github or the forum.

Added Open/Save Document Commands

From the (new) File menu you can now save and load the entire workspace. This includes not only all current data from tests but also includes (embedded) all opened tests in each Get list.

So, if you have a Spectrum and Intermodulation plot in your workspace both of them will be saved in the document. If the Spectrum test also has three tests shown in the Get list they will also be embedded into the document for later loading. Try this out!

Added Settings Mini Editor

In order to easily set common settings (as shown in the Settings expander bottom left of the screen) there is a new Pencil button that lets you pop up a short Settings window with common fields to set.

v 1.2.9     Feb 17, 2026

Risk Summary

Should be minimal.

Added Plot Line ordering to the Test menu and Settings

From the test menu you can decide which traces are plotted on top. Choose from Left, Right, Get/Left (Get plots are on top and Left channel is on top), and Get/Right. Applies to all plots.

Added Plot Line thickness setting

The constant thick value of 2.0 was a little thin with my light gray background so it's now a floating point Setting that applies to all plots. Usual values are < 6.

v 1.2.8     Feb 8, 2026

Risk Summary

This release is very high risk. There were some substantial refactorings of mid-level code to help with consistency and for adding more tests. Trying to reduce the code debt.

Added a new Test menu

The new test menu currently contains just the graph popup menu. This lets you do the graph popup options with the keyboard, too.

Renamed Start to Once when Run exists

For the tests which have a run once (Start) option that is now named Once. I think that's more user-friendly since I constantly found myself trying to remember which was which between Run and Start.

Support 4 Data Summary Boxes in Spectrum

I found myself wanting to compare 4 plots a lot. So... This may get expanded to no limit. While there I fixed the boundary to plot line color matching.

Added Output Gain option

If you have an amplifier gain/buffer block at the generator output to enable higher output voltages, enter the gain for that in Settings. This will automatically adjust all operations for expected additional output gain.

v 1.2.7     Feb 1, 2026

Risk Summary

The relay timing speedup works perfectly in my tests but may cause issues on very slow PCs. This release is higher risk than usual thanks to some long overdue refactoring of Actions.

Add Blackman-Harris-7 Window

Blackman-Harris 7 is a nice windowing scheme so it's now available (Blackman-7).

Repair renaming files

With the switch to .ZIP file format (instead of gzip) a bug was introduced causing load to fail if the file was renamed. That has been repaired.

Reduce relay time

Continuing to try to get ultra-stable results while spending as little time waiting as possible. A few empirical tests were done and some relay wait times were decreased by a second+.

Reduce effect when graph limits change

Especially for pin, having the graph readjust a lot was annoying. Now only the axis being changed adjusts. Also, pin is now way more exact and should be far more useful.

More residual upgrade

The scope Residual graph now has a synched axis, like group delay in the Gain test.

 

Customize the Graph Popup menu

The default ScottPlot right-mouse-button graph popup menu wasn't very useful and now it is.

  1. Pin All - pin the range of all of the axes at once
  2. Fit All - do a Fit To Data for all axes at once.
  3. Snapshot - quickly save/get a test to have it immediately display. Snapshots are named Snap1, Snap2, ... and stored in the default data folder (if there is one in Settings) or in Documents if not. Those names are reused each time you run the program so you can persist these files simply by renaming them after closing QA40xPlot.
v 1.2.6     Jan 24, 2026

Risk Summary

Frequency Response tests have had extensive changes. Otherwise, these are primarily esthetic improvements and small additional features so hopefully slight risk.

Add QA40x readings upper right

On the upper right of the screen (if menus are enabled) it now shows the USB DC Voltage and current as well as the (I think) internal temperature. These values only get updated each time a sweep runs.

Add editor for Get plots test info

When you get a saved plot to display there is now an edit button in the left-side GUI. That brings up a dialog that lets you change the plot name so it reads better in the Legend. You can also view/edit the heading and description. A save button was added to allow the changes to go back to the saved plot file. When a saved test is Gotten with a blank Name field, the app uses the file name to identify the lines in the Legend.

Add a default data folder setting

Adds a Settings option to put all your saved tests and configurations into a specific default folder. This does not affect QADefault.cfg, which always comes from your Documents folder unless supplied on the command line at startup. This setting just defines the starting folder when you click a get, save, or load button.

Add a group delay display to impedance and gain tests

This is a very early implementation of group delay. It is always in ms and shows up in a tentative second Y axis.

Add Shift-Click option to legend labels for a legend group

Hold down a shift key while clicking a legend checkbox (in the legend window) and the entire group of lines with the same suffix (after the @) will hide/show. This helps deal with the large number of plot lines in the THD vs Amplitude and vs Frequency tests.

 

v 1.2.5     Jan 18, 2026

Risk Summary

Start, Run, and Stop now work somewhat differently so there may be cases where they fail to enable correctly. Over the last few releases some objects have changed names so it is probably time to delete your default configuration file and start over with factory defaults.

Update cursor value each time test repeats

For tests that run repeatedly, update the mouse cursor display after a test runs. It was requiring a mouse move to update the displayed value.

Add a Residual waveform display to the Scope test

Per user requests, this adds the ability to show the residual time series (data - fundamental) to the plot. It will be listed in the "Get" list on the left where either channel can be shown or hidden and colors may be manually set. This waveform is (currently) calculated by removing all spectral content within a sixth-octave of the fundamental. This uses the known test-channel fundamental frequency if generator 1 is enabled otherwise it reads the fundamental from the spectral peak.

Do not allow test switching during Run

Switching test pages while running a test was never a good idea but there was no program stop. Now it's explicit and you'll get a warning when you try to flip tests during a run.

v 1.2.4     Jan 13, 2026

Risk Summary

The gain curve math updates are problematic. QA40xPlot uses gain curves for auto-attenuation of IMD, Amplitude and Frequency sweeps, and Response. The unit of measure changes on amplitude (voltage) selection runs through every test and should work ok but now that they save/load with configuration there could be issues.

Fixup selection of Transparent in color picker

Transparent color is used as an indicator to use the default color from the color palette for these chart lines so it should be easy to select. This fixes requiring setting it twice to stick.

Improve gain curve frequency selection

The gain curve algorithm, which calculates required attenuation in sweep tests, did some band limiting out of the audio range that could cause issues when testing high frequency response. It now uses at least your specified frequency range for the gain curve test.

Add a continuous mode Run button to the Response/Gain/Crosstalk/Impedance tests

Particularly when using Chirp, which runs quickly, support repeating these tests, which allows for tuning circuits.

Swap Start to Run for spectrum, imd, and scope tests

This makes Run and Start more consistent for the tests. Run == run repeatedly whereas Start = run once

Add F5,F6,F7 accelerators for run/start/stop

Enable keyboard accelerators for run, start and stop. Use Function keys F5,F6,F7 to avoid conflicts with the left hand GUI panel choices.

Support saving and loading amplitude UOM

Untether the amplitude value from the Unit of Measure (maybe temporarily) to allow for saving the Unit.

Improve generator voltage messaging

Add a dBxx display in generator voltage to match Unit when user selects a logarithmic unit.

Upgrade esthetics of voltage unit button

Hopefully it is more obvious and esthetic.
